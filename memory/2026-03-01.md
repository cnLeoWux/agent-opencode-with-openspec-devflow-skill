### 经验教训

#### 1. Skill-only Repo 规范
- **正确做法**：skill 专用 repo 只应该包含 `SKILL.md` 一个文件
- **错误做法**：把整个 workspace 上传到 skill repo
- **OpenSkill 规范**：每个技能是独立目录，`SKILL.md` 是唯一必需文件

#### 2. GitHub Repo 类型区分
| Repo 类型 | 用途 | 应该包含的内容 |
|----------|------|---------------|
| Workspace Repo | 个人工作区 | 完整的开发 workspace |
| Skill Repo | 分享技能 | 只有 `SKILL.md` |

#### 3. ⚠️ 绝不随便删除文件和目录（**重要教训**）
- 在执行 `rm`、`git rm`、`mv` 等删除/移动操作前，**必须先和 user 确认**
- 即使是看起来"无用"的文件，也先挪到 trash，不要直接 rm
- 所有删除操作前：`ls -la` 确认内容 → 和 user 沟通 → 再执行
- **特别注意**：workspace 根目录下的 `develop/`, `memory/`, `.openclaw/` 都是重要目录，不容删除

#### 4. git clean 重要性
- 创建新 repo 前先确认 `git status` 和 `git ls-tree HEAD`
- Push 前用 `git ls-tree HEAD` 检查远程实际内容
- 使用 `.gitignore` 避免意外包含无关文件

#### 5. 检查清单
创建 skill repo 后：
- [ ] `ls -la` 确认只有必要文件（SKILL.md）
- [ ] `git ls-tree HEAD` 检查本地 commit 内容
- [ ] Push 前用 `git remote -v` 确认远程地址正确
- [ ] GitHub 网页确认显示正常
